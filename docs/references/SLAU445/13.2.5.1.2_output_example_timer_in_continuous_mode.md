# 13.2.5.1.2 Output Example – Timer in Continuous Mode

The OUTn signal is changed when the timer reaches the TAxCCRn and TAxCCR0 values, depending on the output mode. An example is shown in Figure 13-13 using TAxCCR0 and TAxCCR1.

<a id="figure-13-13"></a>

```text
title "Output example - Timer in continuous mode"
x-axis "time" 0 --> 135000
y-axis
  primary "Counts" 0 --> 65535
  secondary "Reset/Set" 0 --> 1
series
  name "Continuous mode"
  y-axis primary
  points
    0       0      Interrupt event - TAIFG
    40000   40000  Interrupt event - EQU1
    50000   50000  Interrupt event - EQU0
    65535   65535
    65536   0      Interrupt event - TAIFG
    105536  40000  Interrupt event - EQU1
    115536  50000  Interrupt event - EQU0
    131071  65535
    131072  0      Interrupt event - TAIFG
    
series
  name "TAxR Max (65535)"
  y-axis primary
  points
    0       50000
    135000  50000
series
  name "TAxCCR0 - 50000"
  y-axis primary
  points
    0       50000
    135000  50000
series
  name "TAxCCR1 - 40000"
  y-axis primary
  points
    0       40000
    135000  40000
series
  name "Output Mode 1: Set"
  y-axis secondary
  points
    0       0
    39999   0
    40000   1
    135000  1
series
  name "Output Mode 2: Toggle/Reset"
  y-axis secondary
  points
    0       0
    39999   0
    40000   1
    49999   1
    50000   0
    105535  0
    105536  1
    115535  1
    115536  0
    135000  0
series
  name "Output Mode 3: Set/Reset"
  y-axis secondary
  points
    0       0
    39999   0
    40000   1
    49999   1
    50000   0
    105535  0
    105536  1
    115535  1
    115536  0
    135000  0
series
  name "Output Mode 4: Toggle"
  y-axis secondary
  points
    0       0
    39999   0
    40000   1
    105535  1
    105536  0
    135000  0
series
  name "Output Mode 5: Reset"
  y-axis secondary
  points
    0       1

    135000  0
series
  name "Output Mode 6: Toggle/Set"
```
